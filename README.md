# 遊戲每日與每週任務清單

這是一個可以放在 GitHub Pages 的單頁工具，用來追蹤遊戲每日與每週任務。

## 功能

- 新增、編輯、刪除遊戲
- 每款遊戲可同時記錄每日任務與每週任務
- 每日任務每天重置，每週任務每週一重置
- 每款遊戲可自訂每日重置時間
- 開著網頁時會定期刷新日期與任務狀態；手機切回頁面也會重新檢查
- 顯示本週每日完成幾天，並列出目前漏掉的日期
- 標記哪款遊戲有月卡或通行證
- 記錄月卡到期日，支援「今天購買 +30 天」快速填入，並自動顯示剩餘天數
- 記錄版本、卡池與結束日；卡池可填結束時間並顯示剩餘時間
- 記錄該版本活動是否已完成，並可篩選活動未完成
- 可在卡片上直接切換版本活動完成狀態
- 月卡、版本、卡池剩 3 天內會醒目提示，已過期會標紅
- 切換清爽、夜間、櫻花、電玩霓虹背景主題
- 可分開查看每日未完成、每週未完成、活動未完成、所有未完成、全部、任務全完成
- 手機友善快速按鈕：今日建議、每日、每週、活動
- 今日建議會依每日、活動、每週與快到期資訊排序
- 可讀取同目錄的 `game-data.json`，依同名遊戲或別名匯入版本、卡池和活動資料
- 顯示目前工具版本
- 拖曳排序遊戲，並提供上移/下移按鈕
- 複製目前未完成清單
- 可用 GitHub Gist 做手機和電腦同步
- 可設定開啟時自動下載雲端、修改後自動上傳同步
- 上傳前會檢查雲端資料是否比較新，降低手機和電腦互相覆蓋的風險

## GitHub Pages

把這個資料夾裡的檔案放到 GitHub repo 根目錄，然後到 repo 的 `Settings -> Pages` 啟用 GitHub Pages。

建議設定：

- Source: `Deploy from a branch`
- Branch: `main`
- Folder: `/root`

啟用後網址通常會是：

```text
https://你的帳號.github.io/你的repo名稱/
```

## 雲端同步

同步功能使用 GitHub Gist。你需要在 GitHub 建立一個 fine-grained token 或 classic token，並給它 `gist` 權限。

第一次使用：

1. 在工具的「雲端同步」填入 GitHub Token。
2. Gist ID 留空。
3. 按「上傳同步」。
4. 工具會自動建立 secret Gist，並填入 Gist ID。

手機同步：

1. 手機打開同一個 GitHub Pages 網址。
2. 填入同一個 GitHub Token 和 Gist ID。
3. 按「下載雲端」。

同步設定：

- 「開啟網站時自動下載雲端」會在有 Token 和 Gist ID 時自動抓最新資料。
- 「修改後自動上傳同步」會在新增、編輯、勾選後自動上傳。
- 手機和電腦如果同時開著，不會即時互相推送；切換裝置前可按一次「下載雲端」確認最新。

注意：Token 會保存在該瀏覽器本機。只在自己的裝置使用，不要貼到公用電腦。Secret Gist 不會公開列出或被搜尋，但如果別人拿到網址仍可能看到內容。

## 遊戲資料匯入

網站可以讀取同一個 GitHub Pages 目錄下的 `game-data.json`，依照遊戲名稱套用資料。

範例：

```json
{
  "updatedAt": "2026-06-28T00:00:00+08:00",
  "games": [
    {
      "name": "原神",
      "aliases": ["Genshin Impact", "Genshin"],
      "versionName": "5.7",
      "versionEndDate": "2026-07-29",
      "bannerName": "角色 A / 角色 B",
      "bannerEndDate": "2026-07-15",
      "bannerEndTime": "17:59",
      "versionEventDone": false
    }
  ]
}
```

注意：`name` 要和網站裡的遊戲名稱一致，才會自動套用。
補充：`name` 或 `aliases` 其中一個要跟網站裡的遊戲名稱一致，才會套用。像網站裡寫「星鐵」時，`game-data.json` 可以把正式名稱寫成「崩壞：星穹鐵道」，再把「星鐵」放進 `aliases`。
